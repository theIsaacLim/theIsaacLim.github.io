What even is open source?

September 30, 2018

With Hacktober coming up, I wanted to clarify a few things about open souce and basically publish a guide to open source as a whole. People seem to misunderstand or misinterpret the meaning of the term “open source”. Here’s what it really means in relation to software.

  1. Open source software is free to use

    • This means that open source software is free for consumers to use. It’s really as simple as it sounds

  2. Open source software allows anyone to modify the source

    • This means that anybody can make a copy of the source code and modify it for their own usage without permission from the creators of the source

    • This also means that they can push a new, edited version of the source code back to the original creators, who can choose to merge the new code or not.

  3. Open source software means anyone can distribute it

    • Anyone can distribute open source software in any way, shape or form.

How do I contribute to open source projects?

The easiest way to do this is through Github. There are plenty of far better guides to starting to contribute to open source such as this one by Github themselves.

Are open source projects unsafe?

Actually, they’re safer than most other software. This is because with potentially millions of people reading your code more mistakes can be spotted quickly and easily. This is also why iOS bugs are far more common than Android bugs, with Android being open source

Why should I contribute to open source projects?

  1. It looks good on your resumé

  2. It enhances your programming ability on real world programming projects with the support of professional developers

  3. It helps you network and meet new professionals

How do open source projects make money?

Oftentimes, open source projects come from larger software companies with proprietary software, such as Microsoft or Apple or Google. These projects benefit these companies so they can afford to spend money on them.

Also, a lot of the time, open source projects rely on donations from its clientele to its users, advertising, or premium subscriptions. Essentially, they work quite similarly to regular software in terms of economics
